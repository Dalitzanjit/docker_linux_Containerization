                        <section class="section"><h1>Install Docker Desktop on Windows</h1><p><em class="reading-time">Estimated reading time: 6 minutes</em></p><p>Docker Desktop for Windows is the <a href="https://www.docker.com/community-edition">Community</a> version of Docker for Microsoft Windows.
You can download Docker Desktop for Windows from Docker Hub.</p>

<p><a href="https://hub.docker.com/editions/community/docker-ce-desktop-windows/" class="button outline-btn">Download from Docker
Hub</a></p>

<p>By downloading Docker Desktop, you agree to the terms of the <a href="https://www.docker.com/legal/docker-software-end-user-license-agreement" target="_blank" rel="noopener" class="_">Docker Software End User License Agreement</a> and the <a href="https://www.docker.com/legal/data-processing-agreement" target="_blank" rel="noopener" class="_">Docker Data Processing Agreement</a>.</p>

<h2 id="system-requirements">System requirements</h2>

<p>Your Windows machine must meet the following requirements to successfully install Docker Desktop.</p>

<h3 id="hyper-v-backend-and-windows-containers">Hyper-V backend and Windows containers</h3>

<ul>
  <li>
    <p>Windows 10 64-bit: Pro, Enterprise, or Education (Build 17134 or higher).</p>

    <p>For Windows 10 Home, see <a href="#system-requirements-for-wsl-2-backend">System requirements for WSL 2 backend</a>.</p>
  </li>
  <li>Hyper-V and Containers Windows features must be enabled.</li>
  <li>
    <p>The following hardware prerequisites are required to successfully run Client
Hyper-V on Windows 10:</p>

    <ul>
      <li>64 bit processor with <a href="https://en.wikipedia.org/wiki/Second_Level_Address_Translation" target="_blank" rel="noopener" class="_">Second Level Address Translation (SLAT)</a></li>
      <li>4GB system RAM</li>
      <li>BIOS-level hardware virtualization support must be enabled in the
BIOS settings.  For more information, see
<a href="/docker-for-windows/troubleshoot/#virtualization-must-be-enabled">Virtualization</a>.</li>
    </ul>
  </li>
</ul>

<h3 id="wsl-2-backend">WSL 2 backend</h3>

<ul>
  <li>Windows 10 64-bit: Home, Pro, Enterprise, or Education, version 1903 (Build 18362 or higher).</li>
  <li>Enable the WSL 2 feature on Windows. For detailed instructions, refer to the
  <a href="https://docs.microsoft.com/en-us/windows/wsl/install-win10" target="_blank" rel="noopener" class="_">Microsoft documentation</a>.</li>
  <li>
    <p>The following hardware prerequisites are required to successfully run
WSL 2 on Windows 10:</p>

    <ul>
      <li>64-bit processor with <a href="https://en.wikipedia.org/wiki/Second_Level_Address_Translation" target="_blank" rel="noopener" class="_">Second Level Address Translation (SLAT)</a></li>
      <li>4GB system RAM</li>
      <li>BIOS-level hardware virtualization support must be enabled in the
BIOS settings.  For more information, see
<a href="/docker-for-windows/troubleshoot/#virtualization-must-be-enabled">Virtualization</a>.</li>
    </ul>
  </li>
  <li>Download and install the <a href="https://docs.microsoft.com/windows/wsl/wsl2-kernel" target="_blank" rel="noopener" class="_">Linux kernel update package</a>.</li>
</ul>

<blockquote>
  <p><strong>Note</strong></p>

  <p>Docker supports Docker Desktop on Windows for those versions of Windows 10 that are still within <a href="https://support.microsoft.com/en-us/help/13853/windows-lifecycle-fact-sheet" target="_blank" rel="noopener" class="_">Microsoft’s servicing timeline</a>.</p>
</blockquote>

<h3 id="whats-included-in-the-installer">What’s included in the installer</h3>

<p>The Docker Desktop installation includes <a href="/engine/">Docker Engine</a>,
Docker CLI client, <a href="/compose/">Docker Compose</a>,
<a href="/notary/getting_started/">Notary</a>,
<a href="https://github.com/kubernetes/kubernetes/">Kubernetes</a>,
and <a href="https://github.com/docker/docker-credential-helpers/">Credential Helper</a>.</p>

<p>Containers and images created with Docker Desktop are shared between all
user accounts on machines where it is installed. This is because all Windows
accounts use the same VM to build and run containers. Note that it is not possible to share containers and images between user accounts when using the Docker Desktop WSL 2 backend.</p>

<p>Nested virtualization scenarios, such as running Docker Desktop on a
VMWare or Parallels instance might work, but there are no guarantees. For
more information, see <a href="/docker-for-windows/troubleshoot/#running-docker-desktop-for-windows-in-nested-virtualization-scenarios">Running Docker Desktop in nested virtualization scenarios</a>.</p>

<h3 id="about-windows-containers">About Windows containers</h3>

<p>Looking for information on using Windows containers?</p>

<ul>
  <li><a href="/docker-for-windows/#switch-between-windows-and-linux-containers">Switch between Windows and Linux containers</a>
describes how you can toggle between Linux and Windows containers in Docker Desktop and points you to the tutorial mentioned above.</li>
  <li><a href="https://github.com/docker/labs/blob/master/windows/windows-containers/README.md">Getting Started with Windows Containers (Lab)</a>
provides a tutorial on how to set up and run Windows containers on Windows 10, Windows Server 2016 and Windows Server 2019. It shows you how to use a MusicStore application
with Windows containers.</li>
  <li>Docker Container Platform for Windows <a href="https://www.docker.com/microsoft/">articles and blog
posts</a> on the Docker website.</li>
</ul>

<h2 id="install-docker-desktop-on-windows">Install Docker Desktop on Windows</h2>

<ol>
  <li>
    <p>Double-click <strong>Docker Desktop Installer.exe</strong> to run the installer.</p>

    <p>If you haven’t already downloaded the installer (<code class="highlighter-rouge">Docker Desktop Installer.exe</code>), you can get it from
 <a href="https://hub.docker.com/editions/community/docker-ce-desktop-windows/"><strong>Docker Hub</strong></a>.
 It typically downloads to your <code class="highlighter-rouge">Downloads</code> folder, or you can run it from
 the recent downloads bar at the bottom of your web browser.</p>
  </li>
  <li>
    <p>When prompted, ensure the <strong>Enable Hyper-V Windows Features</strong> or the <strong>Install required Windows components for WSL 2</strong> option is selected on the Configuration page.</p>
  </li>
  <li>
    <p>Follow the instructions on the installation wizard to authorize the installer and proceed with the install.</p>
  </li>
  <li>
    <p>When the installation is successful, click <strong>Close</strong> to complete the installation process.</p>
  </li>
  <li>
    <p>If your admin account is different to your user account, you must add the user to the <strong>docker-users</strong> group. Run <strong>Computer Management</strong> as an administrator and navigate to <strong>Local Users and Groups</strong> &gt; <strong>Groups</strong> &gt; <strong>docker-users</strong>. Right-click to add the user to the group.
Log out and log back in for the changes to take effect.</p>
  </li>
</ol>

<h2 id="start-docker-desktop">Start Docker Desktop</h2>

<p>Docker Desktop does not start automatically after installation. To start Docker Desktop, search for Docker, and select <strong>Docker Desktop</strong> in the search results.</p>

<p><img src="/docker-for-windows/images/docker-app-search.png" alt="search for Docker app" width="300px" /></p>

<p>When the whale icon in the status bar stays steady, Docker Desktop is up-and-running, and is accessible from any terminal window.</p>

<p><img src="/docker-for-windows/images/whale-icon-systray.png" alt="whale on taskbar" /></p>

<p>If the whale icon is hidden in the Notifications area, click the up arrow on the
taskbar to show it. To learn more, see <a href="/docker-for-windows/#docker-settings-dialog">Docker Settings</a>.</p>

<p>When the initialization is complete, Docker Desktop launches the onboarding tutorial. The tutorial includes a simple exercise to build an example Docker image, run it as a container, push and save the image to Docker Hub.</p>

<p><img src="/docker-for-windows/images/docker-tutorial-win.png" alt="Docker Quick Start tutorial" width="450px" /></p>

<p>Congratulations! You are now successfully running Docker Desktop on Windows.</p>

<p>If you would like to rerun the tutorial, go to the Docker Desktop menu and select <strong>Learn</strong>.</p>

<h2 id="automatic-updates">Automatic updates</h2>

<p>Starting with Docker Desktop 3.0.0, updates to Docker Desktop will be available automatically as delta updates from the previous version.</p>

<p>When an update is available, Docker Desktop automatically downloads it to your machine and displays an icon to indicate the availability of a newer version. All you need to do now is to click <strong>Update and restart</strong> from the Docker menu. This installs the latest update and restarts Docker Desktop for the changes to take effect.</p>

<h2 id="uninstall-docker-desktop">Uninstall Docker Desktop</h2>

<p>To uninstall Docker Desktop from your Windows machine:</p>

<ol>
  <li>From the Windows <strong>Start</strong> menu, select <strong>Settings</strong> &gt; <strong>Apps</strong> &gt; <strong>Apps &amp; features</strong>.</li>
  <li>Select <strong>Docker Desktop</strong> from the <strong>Apps &amp; features</strong> list and then select <strong>Uninstall</strong>.</li>
  <li>Click <strong>Uninstall</strong> to confirm your selection.</li>
</ol>

<blockquote>
  <p><strong>Important</strong></p>

  <p>Uninstalling Docker Desktop destroys Docker containers, images, volumes, and
other Docker related data local to the machine, and removes the files generated
by the application. Refer to the <a href="/desktop/backup-and-restore/">back up and restore data</a>
section to learn how to preserve important data before uninstalling.</p>
</blockquote>
